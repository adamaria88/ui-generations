Saat generate UI prototype Paper.id, hal-hal ini sering meleset dari DS. User minta dijadikan aturan tetap (sesi 2026-05-20, brief Expense Management).

0. **Tabel sticky column** — production Paper.id: **HANYA kolom action (⋮) yang sticky kanan** saat horizontal scroll (ada shadow divider di kiri kolom action). Kolom kiri (checkbox/expand/No.) **TIDAK sticky** — ikut scroll. (Asumsi lama "checkbox sticky kiri" dari ekstraksi sales-receipt SALAH — dikoreksi user via screenshot production. Sudah difix di `design-rules.md` Table List + `style-learner/workflow.md`.)

0b. **Header label + filter row** — production: baris label & baris filter = **satu blok header bertint** (`--color-surface-light-raised`), kontrol filter putih di atasnya. Kontrol filter = ukuran **Aurora form-field** (40px tinggi, padding 8px 12px, radius 4px). **Ikon search di filter input = KANAN** (bukan kiri). Dikoreksi user 2026-05-20 ("color & layout label vs filter harus ikut prod"). Sudah difix di `design-rules.md` Table List.

0c. **Footer pagination** — production: 3 grup (`Jumlah Baris: [N ▾]` · `‹ 1 2 3 4 5 … N ›` · `Menampilkan X hingga Y dari Z entri`) disusun **CENTER sebagai 1 blok** (bukan space-between full-width). Wording wajib: **"hingga"** (bukan "sampai") + **"entri"** generic (bukan nama modul). **Active page** = highlight biru muda ringan (`--color-light-brand-15`) + teks primary — BUKAN solid primary biru + teks putih. **Chevron prev/next** = plain icon transparan tanpa border box. Dikoreksi user 2026-05-20 via screenshot `paper-designer/screenshots/production/sales-invoice/image.png`. Difix di `design-rules.md` Table List + prototype expense-management.

0d. **Bentukan tabel standar (dikunci user 2026-05-20)** — referensi hidup: `_output/expense-management/02-ui.html`. Urutan kolom: checkbox(non-sticky) → expand-chevron(non-sticky) → N kolom data (sortable + filter inline + badge compact + paperclip SVG untuk bukti) → action ⋮ (sticky kanan + shadow divider). Header 2 baris dalam 1 blok bertint (raised), filter inline = Aurora form-field 40px putih, search icon kanan. Scrollbar horizontal `.table-scroll` disembunyikan secara visual (`scrollbar-width:none` + `::-webkit-scrollbar{display:none}`) — tetap bisa scroll. Toolbar list: **TANPA** global search & **TANPA** tombol Filter — filter inline per kolom yang dipakai. Detail lengkap: `design-rules.md` section "Bentukan Tabel Standar".

0e. **State demos (empty/loading/error/no-akses) DI OVERLAY LEVEL, BUKAN in-page** — dikoreksi user 2026-05-20. Prototype Paper.id JANGAN punya tombol/affordance demo state di dalam UI produksi (mis. "Demo: empty state" sebagai ghost button di toolbar) — itu mencemari production look & rancu sama element flow. State-demo harus dipindah ke **overlay** (FAB komen) sebagai menu/picker review-only. Implementasi konkret menyusul; sementara state markup-nya boleh tetap di HTML (dormant) menunggu hook overlay.

0f. **Action menu di tabel WAJIB pakai ikon** — dikoreksi user 2026-05-20 via screenshot production menu Invoice (Kirim/Ubah/Duplikat/Hapus/Tambah Lampiran). Tiap item dropdown 3-dot WAJIB pakai ikon leading (Aurora SVG 18×18, stroke 1.5, round caps). Warna ikon = `--color-text-primary` (dark blue `#133f5d`); item destructive ikon + teks merah `--color-action-destructive-bg`. Menu z-index ≥ 9999 supaya nutup di atas shadow kolom sticky. Detail: `design-rules.md` Table List → "Bentukan Tabel Standar" point 4.

0g. **Dropdown menu di tabel = PORTAL ke `<body>` + position:fixed** — dikoreksi user 2026-05-20 (didiagnose senior tech agent). Naive `position:absolute` ke-clip sama `.table-wrap{overflow:hidden}`. Naive `position:fixed` di tempat (tanpa portal) **rawan containing-block leak** — kalau ada ancestor pakai `transform/will-change/filter/contain:paint` (mis. `#suxc-stage` di flow mode overlay), `position:fixed` jadi relative ke ancestor itu, BUKAN viewport → menu drift/clipped. **Solusi tepat**: pas open, `document.body.appendChild(menu)` (portal), `position:fixed`, anchor pakai `left = btn.getBoundingClientRect().right - menu.offsetWidth` (presisi pixel, lepas dari ambiguity scrollbar). Pakai `documentElement.clientWidth` (BUKAN `window.innerWidth` yang include scrollbar) buat clamp boundary. Auto-flip ke atas kalau kepotong di bottom. Restore parent saat close. Pola ini wajib untuk SEMUA dropdown/popover di dalam container ber-overflow. Detail kode: lihat `_output/expense-management/02-ui.html` function `rowMenu()` + `closeAllRowMenus()`.

0h. **Action Hierarchy = analisis PURPOSE page DULU, baru pilih komponen** — dikoreksi user 2026-05-20 (sesi audit Expense Management). Anti-pattern: langsung pasang `btn--secondary` untuk semua non-primary action → page kelihatan "**ramai aksi**" → primary kehilangan dominance → user bingung. **Cara benar 4 step:** (1) tentukan purpose page satu kalimat. (2) klasifikasi tiap action ke 3 tier: **must-do** (pill primary, MAX 1 per page) / **penting-ga-penting** (text-link, low weight) / **destructive/jarang** (sembunyiin di sub-menu). (3) pilih komponen by tier — `.btn--primary` untuk must-do, `.btn-text` untuk action langsung non-primary (Unduh), `.btn-link` untuk menu trigger non-primary (Tindakan Lainnya, Filter). (4) **squint test**: tutup mata 50%, harus kelihatan 1 titik primary dominan; kalo 2-3 tombol berat sama-sama = hierarchy rusak. **Why:** Pill filled primary itu **anchor visual** — bekerja karena LANGKA. Begitu ada 2-3 pill di page yang sama, dominance hilang. Detail framework: `design-rules.md` section "Action Hierarchy by Page Purpose". Contoh konkret: lihat List Pengeluaran (`_output/expense-management/02-ui.html`) — Tindakan Lainnya (text-link kiri) + Catat Pengeluaran (pill primary kanan), Unduh (text-link in-card toolbar).

0i. **Table card = WAJIB punya search bar di in-card toolbar (kiri) + table-impact action (kanan)** — dikoreksi user 2026-05-20. Earlier rule "toolbar list TANPA global search" SALAH — production Paper.id punya BOTH global search bar (di in-card toolbar atas table) AND inline filter per kolom (di header row). Struktur 3-zone wajib: (1) **Action-bar LUAR card** = menu trigger (Tindakan Lainnya text-link kiri) + primary action (Catat/Buat pill kanan). (2) **In-card toolbar** (di dalam table-wrap, atas table) = search bar pill kiri (`Cari <entity>`, ~320px) + action yang langsung impact table kanan (Unduh text-link, dll). (3) **Table** dengan header bertint + filter inline per kolom. Search di in-card toolbar pakai `.search` (pill with magnifying glass icon kiri, 18px stroke 2). Action di in-card toolbar pakai `.btn-text` (text-link biru primary + icon). Detail: lihat `_output/expense-management/02-ui.html` `.action-bar` + `.tbl-toolbar`.

0j. **Checkbox custom styling WAJIB — jangan native browser** — dikoreksi user 2026-05-20. Native `<input type="checkbox">` punya tinggi/warna inconsistent antar browser & OS; default state (unchecked) ga keliatan jelas → user ga tahu itu checkbox. Aurora pattern: `appearance:none` + custom 18px square, border 1.5px `--color-light-grey-40`, radius 4px, white bg. Hover: border blue primary. Checked: solid blue bg + white ✓ pseudo-element (rotate 45deg). Indeterminate: solid blue bg + white horizontal line. Implementasi: lihat `.chk` di `_output/expense-management/02-ui.html`.

0k. **Klik row di table list = navigate ke View Detail (WAJIB, semua project)** — dikoreksi user 2026-05-20. Setiap list document/item di Paper.id punya detail page; klik row = entry point paling natural (Stripe/Linear/Xero pattern). Tanpa ini, user terpaksa buka 3-dot menu → "Lihat Detail" tiap kali — friction tinggi. **Implementasi:** `<tr>` body → `cursor:pointer` + hover, `onclick="viewDetail(rowId)"`. **EXCLUSION (jangan trigger nav):** klik di checkbox col, expand col, action 3-dot button, dan action menu items — handle via `event.target.closest('.col-chk,.col-exp,.row-act')` di awal handler → return kalo match. ATAU `event.stopPropagation()` di element-element exclusion. **Scope:** Berlaku SEMUA list page semua project (List Invoice, List Pengeluaran, List Customer, dll). Default locked — tidak perlu nanya tiap kali bikin list baru. Detail: [[list-page-default-pattern]] section 4c-bis.

0l. **Placeholder color filter row WAJIB konsisten** — dikoreksi user 2026-05-20. Browser native `<input type="date">` placeholder "dd/mm/yyyy" tampak lebih gelap dari `::placeholder` warna `text-muted` di input text/select — terlihat inkonsistensi visual antar filter. **Solusi:** (1) Date input: `color:text-muted` saat kosong + `:valid {color:text-primary}` saat terisi. (2) Native picker indicator `::-webkit-calendar-picker-indicator{opacity:0}` — hide, overlay Aurora `calendar.svg` via `.col-date` wrapper. (3) Select placeholder via class `is-placeholder` toggle. Hasil: semua placeholder warna sama `text-muted` (input/select/date). **Tinggi filter box = 32px** (compact, locked user 2026-05-20). **Icon WAJIB dari Aurora source** (`icons/assets/search-sm.svg` + `calendar.svg`), bukan ngarang inline SVG. Detail: [[list-page-default-pattern]] section 4b.

1. **Tabel data** — kolom sortable WAJIB ada ikon sort + ada baris **filter inline per kolom** (date/dropdown/search). Bukan header polos.
2. **Pagination** — pakai pola `au-pagination`: dropdown "Jumlah Baris" + chevron first/prev/next/last (18px, `vector-effect:non-scaling-stroke`, disabled = warna muted bukan opacity). Bukan teks "Menampilkan X dari Y" doang.
3. **Ikon** — TIDAK PERNAH emoji (📎📅✓). Selalu Aurora SVG / `au-icon`. Emoji = auto-gagal Penjaga Konsistensi.
4. **Sidemenu active state** — tiap halaman set 1 item sidemenu aktif sesuai modul (`layout-rules.md`: sidemenu konsisten, hanya highlight aktif berubah). Fitur baru tanpa posisi menu → minta user tentukan via komen prototype, jangan biarkan kosong tanpa catatan.
5. **Aset shell** — komponen di `paper-designer/components/` jangan pakai path relatif aset (`../assets/...`); putus saat di-inject ke `_output/<slug>/`. Pakai inline SVG / self-contained. Catatan: kalau file `_output/` sudah ke-inject (SUXC tag hilang), fix shell source TIDAK auto-propagate — harus fix blok baked di file itu juga atau regenerate.

**Why:** Lima ini muncul di review prototype expense-management dan user harus nunjuk satu-satu. Sudah ditulis juga ke Pre-Generation Checklist `paper-designer/rules/design-rules.md` + poin ekstraksi `paper-designer/style-learner/workflow.md`.

**How to apply:** Cek 5 ini sebelum setor prototype apa pun (bagian dari Penjaga Konsistensi). Style Learner juga sekarang mencatat sort/filter/pagination/ikon saat ekstrak production. Lihat juga [[paperverse-design-decisions]] dan [[showcase-audit-status]].
