Manajemen Pengeluaran — User Flow
Expense Management · Paper.id · alur utama (mode cepet) · v1
AktorStaf Finance / Pemilik Bisnis
GoalMencatat & mengelola pengeluaran bisnis supaya rapi, terlacak, dan siap dilaporkan
TriggerAda pengeluaran (beli ATK, bayar listrik, reimburse karyawan) yang harus dicatat
1

User buka menu "Pengeluaran"

Dari sidemenu → Main Area menampilkan halaman daftar Pengeluaran.

Tabel kolom: Tanggal · Kategori · Penerima · Jumlah (right-aligned) · Status · Bukti · Aksi. Search + filter inline per kolom.
Sudah ada pengeluaran tercatat?
TIDAK

Empty state

"Belum ada pengeluaran." Ilustrasi + tombol primary Catat Pengeluaran.

lanjut ke langkah 2
YA

Tabel terisi

User lihat daftar, bisa cari/filter. Lanjut catat baru.

2

User klik "Catat Pengeluaran"

Tombol primary di kanan atas daftar.

3

Main Area berubah jadi Form Catat Pengeluaran

Bukan modal — form replace isi Main Area (Form Page).

Breadcrumb muncul: Pengeluaran > Catat Pengeluaran. Form dibungkus card putih, primary sticky kanan-bawah.
4

User isi data pengeluaran

Tanggal · Kategori · Penerima/Vendor · Jumlah · Metode Bayar · Deskripsi · Upload bukti/struk.

Kategori = Dropdown (≥5 opsi). Bukti/struk = upload opsional. Angka jumlah right-aligned. Field wajib ditandai *.
Field wajib terisi & valid?
TIDAK

Inline error per field

Pesan natural ("Jumlah belum diisi"), input yang sudah diisi dipertahankan. Primary tetap enabled.

balik ke langkah 4 — user perbaiki
YA

Lanjut simpan

Data lolos validasi.

Jumlah ≥ batas butuh persetujuan?
YA

Submit → status "Menunggu Persetujuan"

Pengeluaran masuk antrian approval atasan.

TIDAK

Simpan → status "Tercatat"

Langsung tercatat tanpa approval.

✓ Pengeluaran tercatat di daftar

Toast sukses "Pengeluaran berhasil dicatat" → Main Area balik ke daftar, baris baru muncul dengan status-nya.

User klik "Batal" saat isi form (langkah 4)
ADA PERUBAHAN

Confirmation Modal: "Buang perubahan?"

Default focus = Batal (aksi aman). Lanjut buang → balik ke daftar Pengeluaran.

kembali ke langkah 1 (daftar)
BELUM ADA

Langsung balik ke daftar

Tanpa modal — tidak ada data yang hilang.

kembali ke langkah 1
Langkah user Goal tercapai Exit / gagal (nyambung balik) Titik keputusan
