Input angka dengan format koma (5,000,000) — udah OK?
Setiap input nominal/jumlah pakai format koma otomatis saat ngetik (5,000,000 bukan 5000000). Aurora belum punya komponen dedicated buat ini, jadi gue gabungin komponen input biasa + script kecil buat format-nya. Hasilnya:
Visual
Behavior: user ngetik 5000000 → otomatis berubah jadi 5,000,000 live saat ngetik. Pas submit, koma dihapus lagi jadi angka biasa buat server.
Apa yang gue tanya: ini cara gue (gabung 2 komponen + script) udah jadi "aturan resmi" di catatan rules. Tapi gue mau confirm — visualnya OK, behaviornya OK, atau ada yang mau diubah? Misalnya:
  • Mau format pake titik (5.000.000) gaya Indonesia, bukan koma?
  • Threshold kapan pake separator (sekarang gue bilang "kalau angka >100 baru pake separator")?
  • Boleh negatif (-100,000) atau ga pernah?
→ Komen: setuju format koma + behavior, atau ada yang mau direvisi.

Smooth Transitions
Locked

Snapshot Dirty State
Locked

Pakai saat: bikin form Create/Edit dengan cancel/submit handler.
Complete-by-Default
Locked
